new & delete

new和delete是用于动态内存管理的两个重要运算符。它们分别用于在堆上分配和释放内存，同时也负责调用对象的构造函数和析构函数。

  • new用于分配内存并调用构造函数初始化对象。
  • delete用于释放由new分配的内存，并调用析构函数清理资源。
  • 使用new分配的每块内存都应该用对应的delete释放，以避免内存泄漏

new操作符是C++语言内置的，用于分配内存并调用构造函数来创建对象。当你使用new时，它会执行两个主要步骤：

  1. 调用operator new函数来分配足够的内存。
  2. 在分配的内存上调用对象的构造函数来初始化对象。

例如，以下代码使用了new操作符来创建一个string对象：

std::string* ps = new std::string("Hello, World!");

区别于C中的malloc和free malloc和free是C语言中用于动态内存分配的函数,

malloc和free：

  • malloc用于分配一定大小的内存块，并返回一个指向该内存的指针。它不会调用构造函数，所以不会初始化对象。
  • free用于释放malloc分配的内存。它不会调用析构函数，所以不会清理对象。
  • malloc的返回类型是void*，所以在使用前需要强制转换到适当的类型。

new和delete：

  • new用于分配内存并自动调用构造函数来初始化对象。
  • delete用于释放new分配的内存，并自动调用析构函数来清理对象。
  • new返回的是正确的类型指针，不需要类型转换。

区别：

  1. 类型安全：new是类型安全的，会返回正确的类型指针；而malloc返回void*，需要强制类型转换。
  2. 初始化：new会自动调用构造函数初始化对象；malloc仅分配内存，不初始化。
  3. 异常处理：new在内存分配失败时会抛出异常；malloc在失败时返回NULL。
  4. 配对使用：使用new分配的内存必须用delete释放；使用malloc分配的内存必须用free释放。
  5. 重载：new和delete可以被重载；malloc和free不能。

operator new&operator delete

先说operator new和new的区别，new和delete是操作符，它们不仅负责内存的分配和释放，还会调用对象的构造函数和析构函数。而operator new和operator delete是全局函数，它们只负责内存的分配和释放，不涉及任何构造或析构过程。简单来说，当你创建和删除一个对象时：

  • 使用new操作符，它会：

    1. 调用operator new来分配内存。
    2. 在分配的内存上调用构造函数来初始化对象。
    3. 返回一个指向新对象的指针。

  • 使用delete操作符，它会：

    1. 调用对象的析构函数。

    2. 调用operator delete来释放内存。

因此，new和delete操作符与operator new和operator delete函数的主要区别在于对象的生命周期管理。new和delete操作符会管理对象的创建和销毁，而operator new和operator delete函数只管理内存。123

这也是为什么你可以重载operator new和operator delete来自定义内存分配和释放的行为，但不能重载new和delete操作符本身。它们的行为是固定的，总是包括调用构造函数和析构函数。

operator new是一个用于内存分配的函数。当你使用new关键字创建对象时，new操作符会调用operator new函数来分配足够的内存。这个函数通常返回一个指向未初始化内存的void*指针。operator new可以被重载，这意味着你可以自定义内存分配的行为。

此外，还有一种特殊的operator new称为placement new，它允许在已经分配的内存上构造对象。

需要注意的是，显式调用析构函数通常只在特殊情况下使用，例如在使用placement new时，或者在使用自定义内存管理策略时。在大多数情况下，应该避免显式调用析构函数，因为它可能导致对象被销毁两次，从而引发未定义行为。

​ 如果使用placement new在预分配的内存上构造了一个对象，你需要在不再需要该对象时显式调用其析构函数来销毁它，然后释放内存。这是因为placement new不会在对象的作用域结束时自动调用析构函数。

如果使用placement new和显式调用析构函数的组合来管理对象的生命周期，那么当对象的作用域结束时，编译器不会再次自动调用析构函数。这是因为编译器只会自动调用那些以普通方式（非placement new）构造的对象的析构函数。

当手动调用析构函数后，对象已经被销毁，所以编译器不会再尝试销毁它。因此，不会出现二次销毁的情况。但是需要确保在调用析构函数之后，也使用正确的方法释放内存（例如，如果使用operator new分配内存，则应使用operator delete释放内存）。

这种手动管理对象生命周期的方法需要谨慎使用，因为如果忘记调用析构函数或释放内存，就可能导致资源泄漏或其他未定义行为。所以，这通常只在特殊情况下使用，比如在自定义内存管理或优化性能时。

折叠表达式

折叠表达式是 C++17 中引入的，它允许我们对可变参数模板中的参数包进行操作。参数包就是那些用省略号（...）表示的一系列参数。

想象一下，你有一堆书要叠起来。你可以从左到右（左折叠），也可以从右到左（右折叠）开始叠。在 C++ 中，这个“叠书”的动作就是用某个操作符（比如加号 +）来“叠加”参数包中的所有参数。

这里有几个例子：

  • 加法折叠：如果你想把所有的参数加起来，你可以写成 (args + ...)，这就是一个右折叠，因为它从右边开始叠加参数。如果你写成 (... + args)，这就是一个左折叠，因为它从左边开始叠加参数。
  • 逻辑与折叠：如果你想检查一系列布尔值是否都为真，你可以使用逻辑与操作符 && 并写成 (... && args)。这会检查每个参数，只要有一个为假，整个表达式就为假。
  • 逗号折叠：逗号操作符 (,) 可以用来执行一系列操作，然后返回最后一个操作的结果。例如，(cout << args..., endl) 会打印所有参数，然后打印一个换行符。

1、一元右折叠(unary right fold)
　　( pack op … )
　　一元右折叠(E op …)展开之后变为 E1 op (… op (EN-1 op EN))
2、一元左折叠(unary left fold)
　　( … op pack )
　　一元左折叠(… op E)展开之后变为 ((E1 op E2) op …) op EN
3、二元右折叠(binary right fold)
　　( pack op … op init )
　　二元右折叠(E op … op I)展开之后变为 E1 op (… op (EN−1 op (EN op I)))
4、二元左折叠(binary left fold)
　　( init op … op pack )

　　二元左折叠(I op … op E)展开之后变为 (((I op E1) op E2) op …) op EN

语法形式中的op代表运算符，pack代表参数包，init代表初始值。
初始值在右边的为右折叠，展开之后从右边开始折叠。而初始值在左边的为左折叠，展开之后从左边开始折叠。
不指定初始值的为一元折叠表达式，而指定初始值的为二元折叠表达式。
当一元折叠表达式中的参数包为空时，只有三个运算符（&& || 以及逗号）有缺省值，其中&&的缺省值为true,||的缺省值为false，逗号的缺省值为void()。

左值和右值

在C++中，左值（lvalue）和右值（rvalue）是与表达式相关的概念，用于描述表达式的属性。简单来说：

  • 左值（lvalue）是一个标识符，它表示一个具体的内存位置。左值可以出现在赋值语句的左边或右边，可以取地址，并且具有持久性。例如，变量、数组元素、返回左值引用的函数调用等都是左值。
  • 右值（rvalue）是一个临时值，它没有持久的内存位置。右值通常出现在赋值语句的右边，不能取地址。例如，常量、字面量、临时对象等都是右值。

在C++11之后，右值引入了移动语义和右值引用。右值引用允许我们对右值进行更高效的操作，例如移动语义可以将资源从一个对象转移到另一个对象，而不是进行昂贵的复制操作。

可以将左值引用和右值引用看作是引用传递的两种方法，它们针对不同的场景和需求提供了不同的引用方式。左值引用用于传递可修改的对象，而右值引用用于支持移动语义和临时对象的操作。

移动语义

学习之前可以先复习一下深拷贝

复制(深拷贝与浅拷贝)及拷贝构造 | HahのBlog (rhahr.top)

移动语义本质上是通过改变资源的所有权来避免内存的深度拷贝，和浅拷贝很像很像，但是他们的主要区别：

所有权：浅拷贝不改变对象的所有权，而移动语义会将所有权从源对象转移到目标对象。(窃取)

副作用：如果我们多次进行浅拷贝，不会对源对象产生任何影响，因为它不改变对象的所有权。然而，由于移动语义会改变对象的所有权，所以在移动操作后源对象就不能再被使用了。

性能：在某些情况下，如处理大量数据或实现某些数据结构（如std::vector）时，移动语义由于避免了不必要的拷贝，所以它的性能通常优于浅拷贝。

  • 移动语义是C++11引入的一种优化策略，其背后的目标是消除在复制大对象时可能出现的开销。它允许我们将正在其生命周期结束时的资源(通常是临时对象，也被称为右值)的所有权转移到另一个对象，而不是通过复制来新建资源。

  • 我们使用移动构造函数来实现这种优化。移动构造函数接受一个右值引用，并将资源从源对象”移动”到目标对象，这避免了创建并初始化新对象的开销。在资源被移动之后，源对象会被析构。

总的来说，尽管他们在某种程度上都是在复制数据，但是移动语义和浅拷贝在所有权、副作用和性能等方面都有本质的区别。

单例模式

单例模式是一种常见的设计模式，它确保类只有一个实例，并提供一个全局访问点以访问该实例,是一种组织一堆全局变量和静态函数的方式

  1. 全局访问点（Global Access Point）：单例模式提供了一个全局访问点，使得任何地方都可以方便地访问该类的唯一实例，而不需要知道实例是如何创建的。
  2. 节省资源：单例模式可以节省系统资源，因为它确保一个类只有一个实例存在，避免了重复创建实例所消耗的内存和其他资源。
  3. 全局状态管理：单例模式适用于需要全局状态管理的情况，例如日志记录器、配置管理器等。
  4. 惰性初始化：单例模式可以延迟实例化，即在需要时才创建实例，而不是在程序启动时就创建。这种惰性初始化的方式可以节省资源并提高程序的性能。
  5. 线程安全性：单例模式的实现可以确保在多线程环境下只有一个实例被创建，并且提供了线程安全的访问方法，避免了多线程环境下的竞态条件和数据竞争问题。

在C++中，单例模式通常通过以下方式实现：

  1. 私有化构造函数（Private Constructor）：将类的构造函数私有化，这样外部就无法通过构造函数创建类的实例。
  2. 静态成员变量（Static Member Variable）：在类中定义一个静态成员变量，用于存储唯一的实例。
  3. 静态成员函数（Static Member Function）：提供一个静态成员函数来获取类的实例。这个函数通常会检查实例是否已经存在，如果不存在，则创建一个新实例并返回，如果已经存在，则直接返回该实例。

文件描述符

文件描述符（File Descriptor，简称FD）是操作系统中用于访问文件和I/O资源的抽象表示。在Unix和类Unix系统中，文件描述符是一个整数，它唯一地标识了进程已打开的文件、管道、网络连接等资源。这里的“文件”不仅指传统意义上的文件，还包括任何可进行I/O操作的资源。

文件描述符的基本概念

  • 整数索引：文件描述符是一个非负整数。当打开一个文件时，系统会返回一个文件描述符，用于后续的读写操作。
  • 标准输入输出：程序启动时，默认会打开三个文件描述符：0（标准输入stdin）、1（标准输出stdout）、2（标准错误stderr）。
  • 资源引用：文件描述符提供了一个统一的接口来引用各种类型的I/O资源，如普通文件、目录、套接字等。

文件描述符的工作原理

当一个进程打开或创建文件时，内核会创建一个文件描述符，并将其返回给进程。这个描述符实际上是一个索引，指向内核中的一个文件表项。这个表项包含了文件的各种信息，如当前的读写位置、文件的状态和访问权限等。

文件描述符的存储和管理

  • 进程文件描述符表：每个进程都有一个文件描述符表，用于跟踪该进程打开的所有文件描述符。
  • 系统级文件表：系统维护一个全局的文件表，记录所有打开的文件的状态和属性。
  • inode表：文件系统维护一个inode表，存储文件的元数据，如文件大小、权限、创建和修改时间等。

文件描述符的使用场景

  • 文件操作：通过文件描述符，进程可以对文件执行读写、定位、锁定等操作。
  • 网络通信：套接字也会被分配文件描述符，进程可以通过它来进行网络数据的发送和接收。
  • 进程间通信：管道和FIFO等IPC机制也使用文件描述符来传递数据。

文件描述符的限制

  • 数量限制：每个进程能打开的文件描述符数量是有限的，这个限制可以通过系统调用或配置文件来调整。
  • 生命周期：文件描述符在进程生命周期内有效，当进程终止时，它打开的所有文件描述符都会被自动关闭。

文件描述符是Linux/Unix系统编程中的一个核心概念，理解它对于进行系统级编程至关重要。它提供了一个简洁的抽象，使得对各种类型的I/O资源的操作变得统一和简单

流(stream)

在 C++ 中，流（stream）是一个抽象概念，表示数据的流动或者流向。流可以是输入流（input stream）或输出流（output stream），用于从某个源头读取数据或者向某个目的地写入数据。

流的概念源自于 I/O 设备（如键盘、屏幕、文件等）之间的数据传输过程。流将数据抽象为一系列的字符或字节，通过特定的方式从源头流向目的地。流可以是单向的（只能进行输入或输出）或者双向的（既可以进行输入也可以进行输出）。

在 C++ 中，流被实现为一系列的类，分别用于处理不同类型的数据流。常见的流类包括：

  2. 输出流类（output stream classes）：用于向目的地写入数据，例如屏幕、文件等。常见的输出流类有 std::ostream、std::ofstream 等。
  3. 文件流类（file stream classes）：用于文件的输入输出操作。常见的文件流类有 std::ifstream、std::ofstream、std::fstream 等。

这些流类提供了一系列的成员函数和操作符，用于从流中读取数据或者向流中写入数据。通过使用这些流类，你可以方便地进行文件操作、标准输入输出、字符串流等各种输入输出操作。

流，字如其名，像是水流，cout，cerr以及自定义的流file这些看作水流本身，流式输入输出由运算符<</>>来指定水流方向。

创建一个流式对象，例如输出文件流对象ofstream outFile ，对象outFile本身就是一个水流，当其与任意文件关联，也就是outFile.open("filename")，之后的操作都是对关联的文件的操作，outFile << "Hello, file!"; 像是将字符串放在水流中outFile中，因为outFile关联了文件filename，所以水流的终点就是这个文件内部。

水流并不是畅行无阻的，有时候中间会有水坝的存在(缓冲机制)，正常情况下，水（数据）会在水坝（缓冲区）中积累，直到达到一定的量，然后水坝会放水（刷新缓冲区），将积累的水（数据）一次性地流向下游（屏幕或文件）。

IO操作是消耗性能的，缓冲机制就是减少IO的次数，它可以减少对下游（屏幕或文件）的写入操作次数，从而提高效率。如果没有缓冲区，每次向 std::cout 添加数据都会立即写入屏幕，这会导致大量的I/O操作，降低程序的性能。

C++中存在流操纵器用于控制输入/出流，一个常见的 unitbuf 将导致输出在每次插入后进行刷新。也就是数据生成即立即输出，不会被缓存，使用 std::unitbuf 会启用 unitbuf 标志，如同调用 str.setf(std::ios_base::unitbuf)。相反，使用 std::nounitbuf 会禁用 unitbuf 标志，如同调用 str.unsetf(std::ios_base::unitbuf)

使用：std::cout << std::unitbuf; // 启用自动刷新

手动打开水坝QAQ，例如，如果你正在写一个需要实时日志记录的程序，你可能会希望日志信息能够立即显示在屏幕上或写入文件，而不是在程序崩溃或异常退出时可能丢失未刷新的数据

在这个程序中，我们有三种不同的输出方式：

  1. 第一种情况，我们只是简单地使用 std::cout 输出文本，没有立即刷新。你会注意到，“开始计算…” 和 “计算完成。” 几乎同时出现，因为在耗时操作期间，输出被缓存了。
  2. 第二种情况，我们在输出 “开始计算…” 后立即使用 std::flush 刷新缓冲区。这会导致 “开始计算…” 立即出现，然后程序会等待3秒，最后输出 “计算完成。”。
  3. 第三种情况，我们使用 std::endl 来结束输出。std::endl 会插入一个换行符并刷新缓冲区，所以 “开始计算…” 会立即出现，然后程序等待3秒，最后输出 “计算完成。”。

标准输出流cout就有缓冲机制而标准错误流cerr则是立即刷新，缓冲区刷新还有其他情况：

  • 缓冲区满时。
  • 程序正常结束时，标准库会清理所有的I/O流，包括刷新 std::cout 的缓冲区。
  • 当你输出一个换行符（'\n'）时，如果 std::cout 被设置为行缓冲模式，它会刷新缓冲区。这是很多环境的默认行为，尤其是当输出到终端时。
  • 使用流操纵符 std::endl，它会插入一个换行符并刷新缓冲区。
  • 显式调用 std::flush，它会刷新缓冲区而不输出任何字符

输出流对象是字节的目标。 三个最重要的输出流类为 ostream、ofstream 和 ostringstream。

ostream 类通过派生类 basic_ostream 支持预定义的流对象：

  • cout 标准输出
  • 具有有限缓冲的 cerr 标准错误
  • clog 类似于 cerr，但具有完全缓冲

很少从 ostream 构造对象；通常使用预定义的对象。 在某些情况下，可以在启动程序后重新分配预定义的对象。 可配置为缓冲或无缓冲操作的 ostream 类最适合于连续的文本模式输出。 基类 ios 的所有功能包含在 ostream 中。 如果构造 ostream 类的对象，则必须指定构造函数的 streambuf 对象。

ofstream 类支持磁盘文件输出。 如果需要一个只输出的磁盘，请构造 ofstream 类的对象。 在构造 ofstream 对象时或在调用该对象的 open 成员函数时，可以指定 ofstream 对象是否接受二进制或文本模式数据。 许多格式设置选项和成员函数适用于 ofstream 对象，包括基类 ios 和 ostream 的所有功能。

如果在构造函数中指定文件名，则在构造对象时，将自动打开该文件。 另外，可以在调用默认构造函数后使用 open 成员函数。

类似于运行时函数 sprintf_s，ostringstream 类支持向内存中的字符串进行输出。 若要通过使用 I/O 流格式设置在内存中创建一个字符串，请构造 ostringstream 类的对象。
