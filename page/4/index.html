constexpr

这是C++11 中引入的新关键字，顾名思义：常量(const)+表达式(expr)，也就是说可以用声明一个表达式为常量。看下面这个例子，这样的代码是非法的，因为函数 get_value() 只有在运行期才会被调用，而数组的长度必须是编译期已知的量（当你在栈中为数组申请内存时 它必须是一个编译时就需要知道的常量）

而使用 constexpr 声明函数的话，就可以顺利编译，因为可以在编译期调用常量函数了

讲到这里 constexpr 的含义也就呼之欲出了，constexpr 的含义就是告诉编译器，该表达式的值可以在编译期计算得到，让编译器能够尽情优化。

除了定义函数表达式，constexpr 也可以用于声明常量

终于到 const 出场了，在C++ 中 const 包含了两种含义：常量 和 只读，通过上面的讨论我们明白了 constexpr 也可以表示常量，两者在常量定义上的功能存在交集，在声明常量变量的情况下，两者作用相同。但是为了程序的规范易读，我们不妨做一些限制规范：

- constexpr 用于声明所有的常量以及常量表达式

- const 用于声明变量为只读，不可更改

stack and heap

栈与堆，本质就是内存分配的不同（memory allocation），栈分配的内存空间相邻而堆随机

除了生存周期超出栈的能力范围，或是所需加载空间过大，也就是除了必须使用堆分配的情况下，尽量分配在栈中，在栈里的操作不过是一条cpu指令，效率高

程序执行概述

程序和指令的关系

  • 指令周期：CPU取出并执行一条指令的时间

    时钟周期: 是CPU工作的最小时间单位，也称节拍脉冲或T周期，其值等于机器主频的倒数。

    机器周期: 早期的计算机有机器周期的概念，特指一个指令周期中的不同阶段操作所用时间

    存储周期: 是主存的一个指标，指主存进行连续两次独立的读或写操作所需的最小时间间隔

  • 执行顺序：一般按存放顺序执行，可通过相应的转移指令改变执行流程

  • linux中通过反汇编objdump -d filename可得到汇编代码

  • 对于指令push %ebp指令为 ==01010== 101B，其中高亮部分为push指令操作码，101为EBP编号

一条指令的执行

指令执行过程：

  1. 取指令
  2. PC+ “ 1 ”（1指一条指令的长度，定长指令字每次都相同，变长可能不同）
  3. 指令译码
  4. 主存地址运算
  5. 取操作数
  6. 算术/逻辑运算
  7. 存结果

以上每部都需要检测“异常”及“中断”请求，若有则自动跳转异常处理

  • 寄存器传送语言RTL eg：R[esp] <–R[esp]-4 M[R[esp]]<– R[ebp]

java中重写类方法

重写类方法是指在子类中定义一个和父类或接口中同名、同参数的方法，以覆盖父类或接口中的方法。
重写类方法的目的是为了实现多态，即根据对象的实际类型，调用不同的方法实现，从而实现不同的行为。
例如，在RentableCar类中，我们重写了Car类中的showInfo方法，以显示RentableCar类特有的信息，
而不是Car类通用的信息。当我们使用RentableCar对象调用showInfo方法时，就会执行RentableCar类中
的showInfo方法，而不是Car类中的showInfo方法。这样就实现了多态，即不同类型的汽车对象可以显示不同类型的信息。
• 什么是重写类方法？重写类方法是指在子类中定义一个和父类或接口中同名、同参数的方法，以覆盖父类或接口中的方法。
例如，Car类中有一个showInfo方法，它的定义如下：
public void showInfo() {
System.out.println(“汽车信息：”);
System.out.println(“品牌：” + getBrand());
System.out.println(“型号：” + getModel());
System.out.println(“颜色：” + getColor());
System.out.println(“价格：” + getPrice());
}
这个方法可以显示任何汽车对象的信息。
但是，如果我们想要显示RentableCar对象的信息，我们可能需要显示更多的信息，例如租金。
这时，我们就可以在RentableCar类中重写showInfo方法，它的定义如下：
@Override
public void showInfo() {
System.out.println(“汽车信息：”);
System.out.println(“品牌：” + getBrand());
System.out.println(“型号：” + getModel());
System.out.println(“颜色：” + getColor());
System.out.println(“价格：” + getPrice());
System.out.println(“租金：” + calcRent(1));
}
这个方法可以显示RentableCar对象特有的信息。
注意，我们在方法前面加了一个@Override注解，表示这个方法是重写了父类中的方法。
• 为什么要重写类方法？重写类方法的目的是为了实现多态，即根据对象的实际类型，
调用不同的方法实现，从而实现不同的行为。
例如，我们有一个Vehicle类，它是Car类和Bike类的父类，它有一个run方法，它的定义如下：
public void run() {
System.out.println(“Vehicle is running”);
}
这个方法可以表示任何交通工具运行的行为。
但是，如果我们想要表示Car对象和Bike对象运行的行为，我们可能需要更具体的描述。
这时，我们就可以在Car类和Bike类中重写run方法，它们的定义如下：
@Override
public void run() {
System.out.println(“Car is running fast”);
}

@Override
public void run() {
System.out.println(“Bike is running slow”);
}
这两个方法可以表示Car对象和Bike对象特有的运行行为。
注意，我们在方法前面加了一个@Override注解，表示这两个方法都是重写了父类中的方法。
现在，如果我们有一个Vehicle类型的变量v，它可以指向任何Vehicle子类的对象。
例如：
Vehicle v = new Car();
或者
Vehicle v = new Bike();
当我们使用v调用run方法时，就会执行v所指向对象的实际类型所对应的run方法。例如：
v.run();
如果v指向Car对象，就会输出“Car is running fast”。如果v指向Bike对象，就会输出“Bike is running slow”。这样就实现了多态，
即不同类型的交通工具对象可以表现出不同类型的运行行为。
怎么样才能重写类方法？重写类方法需要遵循一些规则，否则编译器会报错。这些规则包括：
重写方法必须和被重写方法有相同的名称和参数列表。
重写方法不能有比被重写方法更严格的访问权限。
重写方法不能抛出比被重写方法更多或更广泛的异常。
重写方法不能有比被重写方法不同类型或更低级别的返回值。
重写方法必须在子类中定义，并且子子类和父类或接口之间必须有继承或实现的关系。
重写方法可以使用@Override注解来标记，以便编译器检查是否正确地重写了父类或接口中的方法。
