父对象通常指的是在对象的层次结构中处于上一级的对象。在面向对象编程（OOP）的上下文中，父对象具有以下特点：

  • 继承性：子对象（或子类）会继承父对象（或父类）的属性和方法。这意味着子对象可以使用父对象定义的功能，同时还可以有自己的特定功能12。
  • 层次结构：在一个层次结构中，父对象可以有多个子对象，但一个子对象通常只有一个父对象2。
  • 封装性：父对象可以封装通用的属性和方法，子对象则可以扩展或修改这些属性和方法以满足特定需求2。

例如，在Qt框架中，如果你创建了一个QPushButton按钮，并将其放置在一个窗口widget中，那么这个窗口widget就是按钮的父对象。按钮会继承窗口的一些属性，如位置和可见性，同时也可以有自己的属性，如按钮文本和点击事件处理3。

在更广泛的编程语境中，父对象的概念也可以指代给定属性或方法所属的对象4。这有助于组织代码，使其更加模块化和可重用。123

在Qt中，当你为一个对象指定一个父对象后，父对象会负责管理其子对象的生命周期。这是通过Qt的对象树（Object Tree）机制来实现的。具体来说，父对象如何管理子对象的生命周期包括以下几个方面：

  1. 自动内存管理：当你创建一个对象并指定一个父对象时，这个新创建的对象会被自动添加到父对象的子对象列表中。当父对象被销毁时，它会自动销毁其所有的子对象。这样，程序员就不需要手动删除每个子对象，从而避免了内存泄漏的风险1。
  2. 析构过程：在父对象的析构函数中，会调用子对象的析构函数，确保子对象被正确地清理。如果子对象是在堆上分配的（使用new关键字创建的），则会调用delete来释放内存1。
  3. 延迟删除：在某些情况下，如果你需要删除一个QObject对象，推荐使用QObject::deleteLater()方法而不是直接使用delete。这个方法会将删除操作延迟到事件循环的下一个迭代，这通常是线程安全的，并且可以防止在对象正在处理事件时被删除的问题2。
  4. 关闭与删除：默认情况下，当一个QWidget窗口关闭时，它不会被自动删除，只是被隐藏。如果你希望在窗口关闭时自动删除窗口对象，可以在窗口的构造函数中设置Qt::WA_DeleteOnClose属性。这样，当窗口关闭时，Qt会自动调用窗口对象的析构函数来释放资源2。

顶点属性和内存布局

在OpenGL中，理解属性（Attributes）如位置（Position）、纹理（Texture）、法线（Normal）是3D图形编程的基础。这些属性通常用于定义一个物体的几何形状和外观，它们在顶点着色器（Vertex Shader）中被处理，并用于渲染管线中的各种计算，**==顶点一词不只是表示一个点，而是顶点相关的所有属性的集合，也就是说一个顶点有多个属性数据==**。

  • 位置（Position）: 通常是一个三维向量，表示顶点在空间中的坐标。在顶点着色器中，位置会被变换到裁剪空间（Clip Space），以便进行视图和投影变换。
  • 纹理（Texture）: 纹理坐标用于将纹理映射到3D对象上。纹理坐标通常是二维的，与纹理图像的坐标系统对应，用于在片段着色器（Fragment Shader）中采样纹理图像。
  • 法线（Normal）: 法线是垂直于顶点表面的单位向量，用于光照计算。在光照模型中，法线用于确定表面与光源之间的角度，从而计算出正确的光照效果。

在实际应用中，法线贴图（Normal Mapping）技术可以用来增强视觉效果，使得平面表面看起来具有凹凸感。法线贴图通过在每个像素处修改法线向量，模拟出更加复杂的表面细节。这需要在切线空间（Tangent Space）中进行计算，其中切线（Tangent）和副切线（Bitangent）向量与法线向量一起定义了一个局部的坐标系。

在使用法线贴图时，需要计算出切线和副切线，然后构建一个TBN矩阵（Tangent-Bitangent-Normal Matrix），这个矩阵用于将法线从切线空间变换到世界空间或视图空间，以便进行光照计算。这个过程涉及到顶点着色器和片段着色器中的一系列向量和矩阵变换。

总的来说，属性、位置、纹理和法线是3D图形中用于定义物体外观和行为的基本元素，它们在OpenGL渲染管线中起着至关重要的作用。通过理解和正确使用这些概念，可以创建出丰富多彩的3D场景和效果。如果你有更具体的问题或需要示例代码，随时可以问我！

层次结构与继承关系

Qt的类层次结构是一个庞大而复杂的体系，它包含了许多不同的类，这些类通过继承关系相互联系。在Qt中，所有的类都继承自QObject或者其子类。QObject是Qt对象模型的基础，提供了信号和槽机制、对象树结构管理等功能。

主要的模块，它们分别提供不同的功能和类：

  1. QtCore：这是Qt框架的核心模块，提供了非GUI的基础功能，如事件循环、信号和槽机制、基础数据类型、文件操作等。它是其他所有模块的基础。
  2. QtGui：这个模块提供了基础的图形界面功能，如绘图、窗口管理、事件处理等。它建立在QtCore之上，为更高级别的用户界面组件提供底层支持。
  3. QtWidgets：建立在QtGui之上，提供了一套完整的用户界面控件，如按钮、对话框、输入框等。这些控件用于构建传统的桌面应用程序。
  4. QtNetwork：这个模块提供了网络通信的功能，如TCP/IP、UDP等。它也依赖于QtCore模块。
  5. QtSql：提供了数据库操作的功能，支持多种数据库系统。它也是基于QtCore模块。
  6. QtQuick：用于构建现代化的动态用户界面，它使用QML语言来描述界面。QtQuick建立在QtGui之上，并且通常与QtQml模块一起使用。
  7. QtQml：提供了QML语言的解析和执行功能，是QtQuick的基础。
  8. QtMultimedia：提供了音频、视频以及摄像头的支持。

这些模块之间的关系是层次化的，高层模块依赖于低层模块。例如，QtWidgets依赖于QtGui和QtCore，而QtGui则依赖于QtCore。这种层次结构使得开发者可以根据需要选择合适的模块来构建应用程序。

此外，Qt还有其他模块，如Qt3D、QtWebEngine等，它们提供了更专业的功能。每个模块都包含了一系列相关的类，这些类通过继承关系相互联系，共同构成了Qt的类层次结构。

QT事件循环

发表于 2024-03-24 更新于 2024-09-25 分类于 QT
本文字数： 755 阅读时长 ≈ 1 分钟

在Qt中，事件循环（Event Loop）是一个核心概念，它负责接收和分发应用程序中的各种事件。事件循环确保应用程序保持响应，并在适当的时候处理事件。下面是对Qt事件循环的详细讲解：

事件循环的作用

事件循环是Qt事件驱动系统的核心。它运行一个持续的循环，等待事件的发生（如用户输入、定时器触发、网络消息等），并将这些事件分发给相应的对象进行处理。这个机制使得应用程序能够同时处理多种操作，而不会阻塞用户界面。

事件循环的工作流程

  1. 启动事件循环：当你在main函数中调用QApplication::exec()时，事件循环开始运行。这个调用会阻塞，直到事件循环结束，通常是当主窗口关闭时。
  2. 等待事件：事件循环会等待操作系统或者应用程序产生事件。
  3. 事件的分发：一旦事件发生，事件循环会从事件队列中取出事件，并根据事件类型将其分发给相应的事件处理器。
  4. 事件处理：事件处理器（通常是一个对象的成员函数）会处理事件。例如，一个按钮被点击会产生一个点击事件，事件循环将这个事件发送给按钮对象，按钮对象通过调用其clicked()槽函数来响应这个事件。
  5. 继续循环：处理完事件后，事件循环会继续等待新的事件。

自定义事件处理

Qt允许你自定义事件处理器。你可以通过重写QObject的event()函数或者特定事件的处理函数（如mousePressEvent()）来自定义事件的处理方式。

事件循环的结束

事件循环会一直运行，直到调用了QApplication::quit()，这通常发生在主窗口关闭时。调用quit()会导致exec()返回，从而结束事件循环，程序随后退出。

事件循环的重要性

理解事件循环对于编写响应式的Qt应用程序至关重要。它不仅关系到用户界面的响应性，还涉及到程序的正确性和效率。正确地使用事件循环可以避免界面冻结和应用程序崩溃。

std::unordered_map

std::map和std::unordered_map是C++标准库中的两种非常有用的容器，它们都可以存储键值对，但在内部实现和性能特性上有所不同。

std::map:

  • std::map是一个基于红黑树的平衡二叉搜索树实现的关联容器。
  • 它存储的元素是按照键来排序的，因此可以非常快速地进行查找、插入和删除操作。
  • std::map中的键是唯一的，不能有重复的键。
  • 它的主要优点是，可以保证元素按照特定顺序存储，通常是按照键的升序。
  • 查找操作的时间复杂度为O(log n)，其中n是容器中元素的数量。

std::unordered_map:

  • std::unordered_map是基于哈希表实现的关联容器。
  • 它不保证元素的顺序，因此不能依赖于元素的任何排序。
  • std::unordered_map中的键也是唯一的，不能有重复的键。
  • 它的主要优点是，在理想情况下，查找、插入和删除操作的时间复杂度可以接近O(1)。
  • 但是，由于哈希冲突的存在，这些操作的时间复杂度可能会变差，尤其是当哈希函数分布不均匀时。

在选择使用std::map还是std::unordered_map时，需要考虑你的应用场景。如果你需要保持元素的顺序，则应该使用std::map。如果你更关心查找效率，并且不在乎元素的顺序，那么std::unordered_map可能是更好的选择。

std::unordered_map

前面提到了，std::unordered_map就是哈希表，什么是哈希表，看看newbing的解释：

想象一下，你有一个巨大的图书馆，你需要一种方法快速找到任何一本书。如果你把所有的书按照书名的字母顺序排列，那么即使图书馆很大，你也可以通过书名快速找到书。但是，如果每次你想找一本书都要从头到尾查找，那就太慢了。这就是哈希表发挥作用的地方。

哈希表就像是一个有很多小抽屉的巨大书架。每个抽屉都有一个特定的编号，而这个编号是由哈希函数决定的。哈希函数就像是一个魔法转换器，你告诉它你想找的书名，它就给你一个抽屉的编号。这样，你就可以直接去那个编号的抽屉找书，而不用查看每一本书。

你只需要记住这个编号，就可以直接找到书，这样查找速度就非常快了。

但是，有时候会出现这样的情况：两本不同的书名被哈希函数转换成了同一个抽屉编号。这就是所谓的哈希冲突。比如说，”哈利波特”和”指环王”都被转换成了编号5。这时候，我们需要一种方法来解决这个问题，通常的做法是在同一个抽屉里再加一个小盒子，这样”哈利波特”可以放在一个盒子里，”指环王”放在另一个盒子里。

这样，即使两本书的编号相同，你也可以通过小盒子的标记来区分它们。

实际上就是哈希表中的每一个节点是一个键值对，一个键和一个值（key-value pair）通过键映射到值

  • 对于基本数据类型和标准库中的类型，C++标准库提供了默认的哈希函数。这些默认的哈希函数通常是为了提供一个简单且高效的方式来将数据转换成一个整数值，这个整数值就是哈希值。例如，对于整数类型，哈希函数可能就是简单地返回其本身的值；对于字符串类型，哈希函数会计算字符串中字符的某种组合，以生成一个唯一的整数值。
  • 但是，如果你想在std::unordered_map中使用自定义类型作为键，你需要提供一个合适的哈希函数和相等性判断函数。这通常是通过特化std::hash模板和提供operator==来完成的。

操作符”[ ]”

写过数据结构的都知道，常常要为一个操作符或函数提供不同的版本来适配const和非const，但是这个方括号操作符并没有const版本，为什么呢？

operator[]：如果键不存在，它会创建一个新的键值对，并将值初始化为该类型的默认值。例如，如果值类型是int，新创建的键值对的值将被初始化为0。const表示常量不可修改，而”[ ]”本身就是修改的行为，所以没有const版本，对应的容器内有个at方法

.at()：这个方法只用来访问已存在的键值对。如果你尝试使用.at()访问一个不存在的键，它会抛出一个std::out_of_range异常。.at()方法不会创建新的键值对因此，你可以认为.at()是一个更安全的方法，因为它不会意外地插入新的键值对。如果你需要确保不修改unordered_map，应该使用.at()方法。

回到之前的内容

当你向std::unordered_map插入一个键值对或者查找一个键时，发生的步骤大致如下：

  1. 哈希函数接收键作为输入。
  2. 哈希函数计算并返回一个哈希值。
  3. 哈希表根据这个哈希值决定元素应该存储或查找的桶的位置。

这个过程使得查找和插入操作非常快速，因为它避免了遍历整个容器。哈希值是内部使用的，而作为用户，你通常不需要直接处理哈希值，你只需要关心键和值。

方才那个hash模板特化的代码属是抽象， hash<>()() 这这这，大不合适，但是也不理解了，会用即可，

  1. hash<int>()和hash<std::string>()是创建一个std::hash类型的临时对象，这个对象专门用来计算int类型和std::string类型的哈希值。
  2. 紧跟在后面的()是对这个临时对象的函数调用操作符，它将 key.CityName 作为参数传入，计算它们的哈希值。

让我们分解一下这个表达式：

  • std::hash<int>()创建了一个用于计算整数哈希值的std::hash对象。
  • std::hash<int>()(key.CityName)调用这个对象的operator()，传入key.CityName作为参数，返回key.CityName的哈希值。

总之：无论是普通类型还是自定义类型的键，在std::unordered_map中，键都会通过内部的哈希函数转换为哈希值。然后，这个哈希值被用来在哈希表的内部结构中定位和访问对应的值

最后关于解决哈希冲突，开放寻址法（Open Addressing）和链表法（Chaining）是解决哈希表中哈希冲突的两种常见方法。它们的主要区别在于处理冲突的方式不同。

开放寻址法：

  • 当插入的元素发生冲突时，即该位置已被占用，开放寻址法会寻找哈希表中的下一个空闲位置。
  • 实现方式包括线性探测、二次探测和双重散列等。
  • 开放寻址法的优点是实现简单，数据存储在一个连续的数组中，可以利用CPU缓存加快访问速度。
  • 缺点是删除操作复杂，容易产生聚集问题，导致性能下降。

链表法：

  • 每个哈希表的槽位对应一个链表。所有散列值相同的元素都会被添加到相同槽位对应的链表中。
  • 当发生冲突时，元素会被直接添加到链表的末尾。
  • 链表法的优点是处理冲突简单，扩展容易，不会产生聚集问题。
  • 缺点是链表的指针需要额外的内存空间，且链表的遍历可能会因为CPU缓存不命中而较慢。

NULL

在C++中，NULL和nullptr都用于表示空指针，但它们之间存在一些重要的区别：

  • NULL是一个宏，通常被定义为整数0。在C++中，NULL可能会导致类型歧义，因为它可以被视为整数而不仅仅是指针。
  • nullptr是C++11中引入的一个关键字，它的类型是std::nullptr_t。nullptr专门用于表示空指针，它可以隐式转换为任何指针类型，但不会转换为整数类型，这避免了与整数的歧义。

例如，如果你有两个重载的函数，一个接受整数类型参数，另一个接受指针类型参数，使用NULL可能会导致调用整数类型的版本，而使用nullptr则会明确调用指针类型的版本。这就是nullptr相较于NULL的一个优势。

此外，nullptr不仅清晰地表达了程序员的意图，即表示空指针，而且还提供了更好的类型安全性。

额外内容：

  1. Entity() = default;

在C++中，Entity()=default;是C++11标准中引入的一种语法，用于告诉编译器使用默认生成的构造函数。这意味着你不需要为这个构造函数编写任何特定的实现代码，编译器会为你生成一个标准的构造函数。

这个功能特别有用，因为在某些情况下，如果你已经提供了其他构造函数，编译器不会自动生成默认构造函数。使用=default可以确保即使在这些情况下，你也有一个默认的构造函数。

此外，使用=default可以保持类的某些属性，比如让类保持为聚合类型或者平凡类型。如果你想让你的类保持为聚合类型或者平凡类型，你就需要使用=default，而不是提供一个空的构造函数实现12。

简而言之，Entity()=default;的作用是：

  • 告诉编译器生成默认的构造函数。
  • 确保即使定义了其他构造函数，类也有一个默认构造函数。
  • 保持类的聚合或平凡类型的特性。
  1. offsetof

“offset of”宏在C和C++编程中用来确定结构体(struct)或类中成员变量的内存偏移量。简单地说，它会给出从结构体或类的开始地址到特定成员的字节偏移量。这个宏通常在与底层硬件进行交互、进行内存操作时非常有用，例如，在需要对结构体中的字段进行指针运算时。
