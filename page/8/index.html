time

<chrono>库是一个非常有用的工具。<chrono>库提供了一组强大的时间处理工具，允许你以类型安全和高精度的方式执行时间操作。

Date and time utilities - cppreference.com

这个库引入了三个主要的时间点类型：

  1. std::chrono::system_clock：代表系统时钟，通常是系统范围内的实时时钟。
  2. std::chrono::steady_clock：代表单调递增的时钟，不受系统时钟的影响，适用于计时器和性能测量。
  3. std::chrono::high_resolution_clock：可能是system_clock或steady_clock，具体取决于实现，提供高分辨率的时间测量。

high_resolution_clock 在不同标准库实现之间实现并不一致，尽量不要使用。通常它只是 std::chrono::steady_clock 或 std::chrono::system_clock 的别名——它是 system_clock 时不是单调的（即时间能后退），但具体实现取决于库或配置。例如：对于 gcc 的 libstdc++ 它是 system_clock ，对于 MSVC 它是 steady_clock ，而对于 clang 的 libc++ 它取决于配置。

通常用户应该直接使用 std::chrono::steady_clock 或 std::chrono::system_clock 代替 std::chrono::high_resolution_clock ：对时长度量使用 steady_clock ，对壁钟时间使用 system_clock 。

<chrono>库还提供了不同的时间持续类型，用于表示时间段的各种单位，例如秒、毫秒、微秒和纳秒。这些类型包括：

  • std::chrono::duration：表示时间段的基本类型。
  • std::chrono::seconds：表示秒。
  • std::chrono::milliseconds：表示毫秒。
  • std::chrono::high_resolution_clock::time_point：这是高分辨率时钟的时间点类型。
  • **std::chrono::duration**：这是一个浮点型的时间段类型，用于存储开始和结束之间的时间差。

利用构造和析构来计算function执行时长

在构造函数中，start被设置为当前时间，然后在析构函数中，end被设置为当前时间，计算时间差，并存储在duration中。最后，通过duration.count()获取时间差的秒数，并转换为毫秒进行输出。

这样的计时器可以用于测量代码执行时间，特别是对于性能分析和优化非常有用。

计时器Timer类

  • m_StartTimepoint 是一个 std::chrono::high_resolution_clock 时间点，使用 high_resolution_clock 确保我们有最高的可能时间分辨率。
  • Timer() 构造函数在 Timer 对象创建时被调用，它将 m_StartTimepoint 设置为当前的时间点。
  • ~Timer() 析构函数在 Timer 对象生命周期结束时被调用，它调用 Stop() 函数来计算时间差并进行输出。
  • Stop() 函数首先获取当前的时间点 endTimepoint；然后转换初始时间点和结束时间点为自 epoch 开始的微秒数（即1970年1月1日00:00:00至今的微秒数）；接着，计算这两个时间点的时间差 duration；这个时间差用微秒表示，转换为毫秒存入 ms；最后，将 duration（微秒）和 ms（毫秒）输出。
  • 注意到 Stop() 函数并没有更改 m_StartTimepoint 的值，也就是说，如果在同一个 Timer 对象上多次调用 Stop() 函数，那么每次调用都会输出从对象创建开始至当前调用的时间。

这种使用方式称为“资源获取即初始化”（RAII）模式。

有了这个Timer计时器就可以对代码进行测试，例如测试智能指针的时间损耗，make unique，make shared和new shared之间的时间区别

当lambda表达式捕获了外部变量时，它实际上是一个包含了捕获列表的闭包对象。这个闭包对象不仅包含了函数体，还包含了捕获的变量的副本或引用。因此，它的类型不再是一个简单的函数指针，而是一个类的实例，这个类有一个重载的operator()方法。

由于捕获的外部变量是闭包对象的一部分，这使得闭包对象的类型和函数指针的类型不兼容，因此不能将捕获外部变量的lambda表达式赋值给函数指针。如果需要以类似函数指针的方式使用lambda表达式，可以考虑使用std::function，它是一个模板类，能够存储和调用任何可调用对象，包括函数指针、成员函数指针、以及捕获外部变量的lambda表达式。

.lib和.dll

lib是和dll对应的。lib是静态链接库的库文件，dll是动态链接库的库文件。
所谓静态就是link的时候把里面需要的东西抽取出来安排到你的exe文件中，以后运行你的exe的时候不再需要lib。
所谓动态就是exe运行的时候依赖于dll里面提供的功能，没有这个dll，你的exe无法运行。

.lib

lib文件是一种库文件，它可以分为两种类型：静态库和导入库。

静态库包含了函数的实际代码，可以在编译时直接链接到可执行文件中，不需要依赖其他文件。

导入库只包含了函数在动态链接库（dll）中的地址信息，可以在运行时动态加载dll文件，实现函数的调用。

lib库会做链接过程，但具体的方式取决于它的类型。如果是静态库，那么它会在编译时被链接器合并到可执行文件中，形成一个完整的程序。如果是导入库，那么它会在运行时被操作系统加载到内存中，然后通过地址映射来调用dll中的函数。

.dll

DLL导出函数，可供应用程序调用；
DLL内部函数，只能在DLL程序使用，应用程序无法调用它们

DLL文件作用是: 函数可执行代码

.lib&.dll

C++创建一个动态链接库，编译后会生成两个可用的文件一个是lib文件一个是dll文件，那么这个lib文件是干嘛的呢？
在使用动态库的时候，往往提供两个文件：一个引入库和一个DLL。引入库包含被DLL导出的函数和变量的符号名，DLL包含实际的函数和数据。在编译链接可执行文件时，只需要链接引入库，DLL中的函数代码和数据并不复制到可执行文件中，在运行的时候，再去加载DLL，访问DLL中导出的函数。

  1. Load-time Dynamic Linking 载入时动态链接
    这种用法的前提是在编译之前已经明确知道要调用DLL中的哪几个函数，编译时在目标文件中只保留必要的链接信息，而不含DLL函数的代码；当程序执行时，利用链接信息加载DLL函数代码并在内存中将其链接入调用程序的执行空间中，其主要目的是便于代码共享。
  2. Run-time Dynamic Linking 运行时动态链接
    这种方式是指在编译之前并不知道将会调用哪些DLL函数，完全是在运行过程中根据需要决定应调用哪个函数，并用LoadLibrary和GetProcAddress动态获得DLL函数的入口地址。

lib文件和dll文件的关系如下：

如果一个程序使用了静态库，那么它不需要dll文件，因为它已经包含了所有的函数代码。
如果一个程序使用了导入库，那么它需要dll文件，因为它只有函数的地址信息，需要在运行时加载dll文件才能找到函数的实际代码。
如果一个程序直接使用了dll文件，那么它不需要lib文件，因为它可以通过API函数（如LoadLibrary和GetProcAddress）来获取dll文件中的函数地址

  • dll.lib实际上是一种静态导入库，与这个dll一起使用，包含了.dll的所有的函数，符号的位置，所以可以在编译时链接他们，如果没有这个文件则需要通过函数名来访问dll文件内的函数

补充

vector中的reserve和resize的区别是：

  • reserve是预留内存空间，它只改变vector的容量（capacity），不改变vector的大小（size）。它可以避免多次不必要的扩容和内存拷贝，提高效率。

  • resize是改变vector的大小，它会增加或删除vector的末尾的元素，从而改变vector的容量和大小。它可以直接访问vector中的元素，而不需要调用push_back或insert

vector分配策略

刚开始了解时有个疑问，vector是动态数组，不应该有多少元素就有多大的容量吗，为什么会有空闲呢？

​ vector是动态数组，但是它也有一些内存分配和管理的策略，为了提高效率和性能，避免频繁的内存重新分配和元素拷贝。vector会根据不同的情况，预留一些额外的空间，用于存放新添加的元素，这样就不必为每个新元素重新分配整个容器的内存空间。这些额外的空间就是vector的空闲容量（capacity - size）

这里涉及到一些其他的小东西：

  • vector的扩容机制不一定是每次加1，而是可能按照一定的倍数增长，比如1.5倍或者2倍123。这样可以减少扩容的次数，提高效率。
  • vector的扩容机制也不一定是每次拷贝所有的元素，而是可能使用移动构造函数，把旧的元素移动到新的内存空间45。这样可以避免不必要的拷贝开销，尤其是对于一些不支持拷贝的类型，比如unique_ptr。
  • vector的扩容机制也不一定是每次重新分配一块新的内存，而是可能使用realloc函数，尝试在原来的内存空间后面扩展，如果成功就不需要拷贝或移动元素6。这样可以节省内存和时间，但是这种方法只适用于一些可平凡复制的类型，比如int或char。
